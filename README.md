# 🚀 ProxyIP & DDNS 监控维护中心 (Pro 版)

这是一款基于 **Cloudflare Workers** 开发的全自动 ProxyIP 监控与域名解析（DDNS）维护系统。它集成了实时健康检测、多源 IP 抓取、域名记录克隆、Telegram 实时通知以及可视化 Web 管理面板。

### 🌟 核心功能
-   **多源同步**：支持从本地 IP 库、远程 TXT 接口、以及其他域名的 A/AAAA 记录中自动提取 IP。
-   **健康检测**：自动轮询检测 IP 连通性及延迟，确保域名解析下的每一条记录都是可用的。
-   **自动补全**：当解析记录少于设定的“最小活跃数”时，系统自动从备选库中筛选优质 IP 补齐。
-   **可视化面板**：无需查看代码，通过 Web 页面即可修改配置、管理 IP 库、查看运行日志。
-   **高级通知**：提供详尽的 Telegram 通知报告，包括移除列表、新增列表、在线总数及延迟信息。
-   **诊断工具**：内置后端接口诊断工具，手动测试任意 IP 的连通性。

## 🛠️ 部署指南 (小白必读)

### 第一步：创建 Cloudflare KV 数据库
1.  登录 [Cloudflare 控制台](https://dash.cloudflare.com/)。
2.  点击左侧菜单的 **Workers & Pages** -> **KV**。
3.  点击 **Create a namespace**，命名为 `PROXYIP_STORE`。

### 第二步：部署 Worker 脚本
1.  进入 **Workers & Pages** -> **Create application** -> **Create Worker**。
2.  给 Worker 起个名字，点击 **Deploy**。
3.  点击 **Edit Code**，将本仓库的 `worker.js` 代码全部复制并粘贴进去。
4.  点击右上角 **Save and deploy**。

### 第三步：绑定 KV 数据库
1.  在 Worker 的管理界面中，点击 **Settings** 选项卡。
2.  点击左侧的 **Variables**。
3.  在 **KV Namespace Bindings** 区域点击 **Add binding**。
4.  **Variable name** 必须填写：`PROXYIP_STORE`。
5.  **KV namespace** 选择你第一步创建的那个。
6.  点击 **Save and deploy**。

### 第四步：设置定时触发器 (自动维护)
1.  在 Worker 管理界面中，点击 **Triggers** 选项卡。
2.  在 **Cron Triggers** 点击 **Add Cron Trigger**。
3.  设置时间间隔（推荐每 30 分钟一次：`*/30 * * * *`）。
4.  点击 **Add trigger**。

## 📝 变量配置详解

部署完成后，访问你的 Worker 域名即可打开管理后台。在 **“系统配置”** 标签页中，你需要填写以下参数：

### 1. Cloudflare 核心配置
| 变量名 | 详细说明 |
| :--- | :--- |
| **CF 邮箱** | 你的 Cloudflare 登录邮箱。 |
| **监控域名** | 你要维护的二级域名，例如 `kr.example.com`。 |
| **Zone ID** | 在 Cloudflare 域名概述页面右下角可以看到。 |
| **API Token** | 建议使用具有“DNS 编辑”权限的 Token，或者使用 Global API Key。 |

### 2. 远程数据源 (DDNS 模式)
| 变量名 | 详细说明 |
| :--- | :--- |
| **远程 API 地址** | 系统会抓取这些 URL 里的所有 IP。多个地址用英文逗号 `,` 隔开。 |
| **待克隆解析域名** | 系统会解析这些域名的 A/AAAA 记录并加入备选库。多个域名用逗号隔开。 |

### 3. 运行参数
| 变量名 | 详细说明 |
| :--- | :--- |
| **检测端口** | 你的 ProxyIP 所使用的端口（默认 50001）。系统会根据此端口检测连通性。 |
| **最小活跃数** | 域名解析下最少保持几个 IP 在线。不足时会自动补全。 |
| **检测后端列表** | 用于检测 IP 是否可用的后端接口，例如 `https://check.dwb.pp.ua/check`。支持填入多个，系统会随机轮询。 |

### 4. Telegram 通知配置
| 变量名 | 详细说明 |
| :--- | :--- |
| **TG Token** | 通过 [@BotFather](https://t.me/BotFather) 申请到的机器人 Token。 |
| **TG ChatID** | 你的个人 ID 或频道 ID（可以通过 [@userinfobot](https://t.me/userinfobot) 获取）。 |

## 🚀 使用建议

1.  **初次运行**：保存配置后，点击页面右上角的 **“一键全量维护”**。这会立即触发一次完整的采集、检测和同步流程。
2.  **IP 来源**：如果你有自己搜集的 IP，可以直接粘贴在 **“IP 数据源”** 的本地库中；如果你想白嫖别人的 IP，可以在 **“待克隆解析域名”** 中填入别人的 Proxy 域名。
3.  **日志查看**：如果发现域名没解析，先去 **“运行日志”** 看看哪里报错（通常是 API Token 填错或权限不足）。
4.  **安全提示**：本系统敏感变量在前端保存后会自动隐藏，确保你的 API 信息安全。
